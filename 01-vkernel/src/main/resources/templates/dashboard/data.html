JSONB Data Extensions

vData supports JSONB extensions for any entity via PATCH /api/v1/data/entities/{type}/{id}/extend. Extensions are stored in a JSONB column and schema-free.

Example extension payload:
{ "custom_fields": { "loyalty_tier": "Gold", "credit_limit": 50000, "preferred_currency": "USD" }, "tags": ["key-account", "priority"], "metadata": { "last_synced": "2026-01-15T10:00:00Z", "source_system": "SAP-ERP" } }
